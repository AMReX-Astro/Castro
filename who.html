  • become a developer

    We are always happy to have new developers as part of the Castro team. Fork the Castro git repository on github, make you changes, and issue a pull request against the master github repo. Any level of changes are welcomed: documentation, bug fixes, new test problems, new solvers, ...

  • open development model

    All of Castro's development is done in the public github repository—anyone can see the updates as they are done.
