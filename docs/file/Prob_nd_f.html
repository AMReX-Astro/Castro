ca_initdata subroutine¶

subroutine ca_initdata(level, time, lo, hi, nvar, state, state_lo, state_hi, dx, xlo, xhi)¶

This routine is called at problem setup time and is used to initialize data on each grid.

Note

all arrays have one cell of ghost zones surrounding the grid interior. Values in these cells need not be set here.

INPUTS/OUTPUTS:

Parameters:
  • level [integer] :: amr level of grid
  • time [real] :: time at which to init data
  • lo (3) [integer] :: hi => index limits of grid interior (cell centered)
  • nvar [integer] :: number of state components.
  • state (state_hi(1)-(state_lo(1))+1,state_hi(2)-(state_lo(2))+1,state_hi(3)-(state_lo(3))+1,nvar) [real] :: scalar array
  • dx (3) [real] :: cell size
  • xlo (3) [real] ::
    xhi => physical locations of lower left and upper
    right hand corner of grid. (does not include

    ghost region).

  • hi (3) [integer]
  • state_lo (3) [integer]
  • state_hi (3) [integer]
  • xhi (3) [real]
Use :

castro_error_module, amrex_fort_module (rt() => rt())
Call to:

castro_error()
